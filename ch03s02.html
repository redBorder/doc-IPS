3.2. Inherit policies

As we have seen before, the organization of network elements created in redborder is hierarchical and therefore establishes dependencies between levels and others.
These dependencies establish an inheritance system that affects both the configuration and the policies applied at each of the levels (from higher to lower hierarchy):
  • Root
  • Domain
  • Sensors
  • Group
  • Binding
Sensors -> Tree: hierarchies and dependencies between levels

Note

The inheritance system establishes that the configuration of higher levels affects that of those "children" that are derived or depend on them. In case the configuration of said children is not edited, the system will assign by default the policies, modes and configurations assigned to the "parent" level. The inheritance system is therefore established from the higher levels and will apply to the lower levels that are derived from it.
General Settings
In Tools section we found the option General Settings from which the user can establish the configuration of the general parameters of the Manager.
This is the highest level of configuration (root domain) so the properties defined here will be inherited by the rest of the domains and sensors that are below this level.
General Settings: General Manager configuration or "root"
Inherited setting: "Inherited"
When a domain or sensor is created or edited for the first time, we will notice that some parameters are accompanied by the label "Inherited", this means that they have inherited the values ​​inherited from the general configuration of the Manager or of the domains or sensors that are above the newly created element. To change this setting, the user will need to override that information.
